doxysphinx.resources

doxysphinx.resources¶

The resources module contains classes will do resource provisioning and patching.

Resources are stylesheets, images, javascripts etc. that contemplate the html files.

Classes

CssScoper(css_selector)

Scopes css-stylesheets to a special selector.

DoxygenResourceProvider(directory_mapper)

Resource provider that will copy/adapt doxygen html resources to output.

Details

protocol ResourceProvider[source]¶

A resource provider copies/adapts necessary resources (images, stylesheets, etc.) to output.

Classes that implement this protocol must have the following methods / attributes:

provide_resources(resource_root)[source]¶

Provide necessary resources to sphinx output directory.

Can also do postprocessing etc.

Parameters

resource_root (Path) – the root resource input directory (e.g. where the html files are located)

Return type

List[Path]

Returns

A list of resources (their target paths) that were copied/provided. Note that in case of some caching (copy if newer) mechanisms this might return only parts of the resources.

cleanup_resources(resource_root)[source]¶

Clean up provided resources that were copied by provide_resources().

Parameters

resource_root (Path) – the root resource input directory (e.g. where the html files are located)

Return type

List[Path]

Returns

A list of resources (their target paths) that were cleaned up/removed.

class DoxygenResourceProvider(directory_mapper)[source]¶

Resource provider that will copy/adapt doxygen html resources to output.

Resource are e.g. stylesheets, images, javascript files etc.

provide_resources(resource_root)[source]¶

Copy doxygen html resource files (see GLOB_PATTERN below) to sphinx output.

The content in the raw html directives can then access these directly.

Return type

List[Path]

cleanup_resources(resource_root)[source]¶

Clean up any provisioned resources that were copied to sphinx output.

Return type

List[Path]

class CssScoper(css_selector)[source]¶

Scopes css-stylesheets to a special selector.

This is done with the help of dartsass (a sass processor).

scope(stylesheet, additional_css_rules=None, scss_patch_callback=None)[source]¶

Scopes a stylesheet to given selector.

The process is as follows: The original stylesheet will be moved to .original.scss. Then an scss file (same name like stylesheet) that will import that .original.scss file will be created. This will be compiled to the same filename as the original stylesheet with the help of dartsass.

Parameters

  • stylesheet (Path) – The stylesheet to scope.

  • additional_css_rules (Optional[List[str]]) – Additional css rules to inject.

  • scss_patch_callback (Optional[Callable[[Path], None]]) – A callback that will be called on the original file

Return type

Path

Returns

The path to the written stylesheet (should be identical to stylesheet).
