doxysphinx.writer

doxysphinx.writer¶

The writer module contains classes that write the docs-as-code output files.

Classes

Details

protocol TocGenerator[source]¶

TocGenerator protocol.

Gets the source_dir (with the html sources) during init and each file to possibly generate a toctree directive for in the generate_toc_for() method. The implementer has then to choose how to implement the toc generation.

Classes that implement this protocol must have the following methods / attributes:

generate_toc_for(file)[source]¶

Generate a toctree directive for a given file.

Parameters

file (Path) – the file to generate the toctree directive for

Return type

Iterator[str]

Returns

a string iterator representing the lines forming the toctree directive

class DoxygenTocGenerator(source_dir)[source]¶

A TocGenerator for doxygen.

Will read the menudata.js to check whether a toctree directive needs to be generated or not.

generate_toc_for(file)[source]¶

Generate a toctree directive for a given file.

Note that the toctree will only be generated when the file is part of a menu structure. :type file: Path :param file: the file to generate the toctree directive for :rtype: Iterator[str] :return: a string iterator representing the lines forming the toctree directive

protocol Writer[source]¶

Protocol representing a Writer that write docs-as-code files.

Classes that implement this protocol must have the following methods / attributes:

write(parse_result, target_file)[source]¶

Write a parsed html result to a target file.

The format of that file is controlled by the concreate Writer implementation.

Parameters

  • parse_result (HtmlParseResult) – The result of a previous html parser run

  • target_file (Path) – The target file to write

Return type

Path

Returns

The written file (should be always identical to target_file input, but allows chaining…)
